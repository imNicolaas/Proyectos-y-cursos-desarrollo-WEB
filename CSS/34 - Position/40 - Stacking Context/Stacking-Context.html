Stacking Context CSS

Hace referencia al algoritmo o la forma que tenemos de entender cómo el navegador va a colocar o "Apilar" las cajas que se vayamos ubicando

  • Documentacion oficial: "Staking Context".

El staking context sigue un orden para darle mas o menos prioridad a los diferentes elementos de la pagina, este es el orden de mayor a menor prioridad:

  1. Elementos posicionados con un z-index de 1 o más

  2. Elementos posicionados sin z-index declarado (o z-index auto)

  3. Elementos no posicionados

  4. Elementos con z-index negativo

Ejemplo:

por defecto y si los elementos nos tiene propiedades declaradas que les den mas o menos prioridad el orden en el que se apilan es el que este en en HTML.

Pero haciendo uso de la segunda propiedad y si a los elementos que estan arriba le damos un tipo de atributo position, este se pondra por arriba y sin necesidad que usar un z-index

y si un elemento se posiciona por encima de otro que es un contenedor, solo se pondra por encima del contenedor en si, los hijos que pueda tener este contendor no se veran afectados. Como se ve con el box 2 estando encima de el parrafo 1 pero no de sus hijos. para controlar esto se puede dar el atributo position con cualquiera de sus variantes al elemento que se quiera tener por delante ya sea el container o el box 2

Si los dos elementos tiene el atributo position otra vez estaran al mismo nivel por lo que el staking context sera el por defecto

Diferentes contextos de Apilamiento:

Si un elemento contenedor tiene dentro elementos HTML se generara un contexto de Apilamiento diferente solo dentro del contenedor, ya que el contendor se posiciona y se apila respecto a lo que le rodea pero dentro de el los elementos hacen lo mismo, y por lo tanto hay otro contexto de Apilamiento.
